# Project Vision

## Overview

**Retro Pong Arcade** is the first project in a series of retro game ports developed in Processing.

The purpose of this repository is not only to recreate a classic arcade experience, but also to build a **portfolio-quality project** with a clear architecture, progressive documentation, and reusable development practices.

This project is intended to serve three roles at the same time:

- a playable game
- a technical portfolio artifact
- a pedagogical resource for teaching game development

## General Goal

Rebuild the classic Pong game using a modular and well-documented architecture, so that the project can later serve as a foundation for future retro ports.

## Specific Goals

- Implement a structured game loop
- Use delta time for frame-independent motion
- Model game flow through states
- Build clear and reusable entities
- Document both technical and conceptual decisions
- Organize development as progressive learning milestones

## Development Approach

The project is being built incrementally. Each stage introduces a new layer of complexity while preserving readability and maintainability.

The first stages focus on:

- project structure
- game core
- time system
- state transitions
- player paddle
- moving ball

Subsequent stages will incorporate:

- collisions
- game rules
- scoring
- CPU opponent
- polishing
- final documentation

## Long-Term Value

This repository should become:

- a reference implementation of Pong in Processing
- a reproducible portfolio pattern
- the architectural seed for future ports such as Breakout, Space Invaders, or Frogger
